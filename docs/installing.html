CVRDecode Installation

There are three options for installing CVRDecode:

  • from Debian packages.
  • using flatpak.
  • using the installer program.
Software documentation is available in the overview.

Installing from Debian packages

First add the Debian repository that contains the libbzdev-java and cvrdecode packages. Instructions are available at https://billzaumen.github.io/bzdev/ .

Next install the default Java for your system and run the following commands (as needed - some packages may already be installed): 


  sudo apt-get install libjcommander-java
  sudo apt-get install libzxing-java
  sudo apt-get install libzxing-core-java
  sudo apt-get install libzxing-javase-java
  sudo apt-get install libbzdev-java
  sudo apt-get install cvrdecode

For Linux systems that use the freedesktop.org standards but that are not Debian systems, the package alien can be used to convert Debian packages to other formats. The links to the individual packages that are are needed and not available as RPM packages are

  • the libbzdev-java link .
  • the cvrdecode link .
When these links are visited, the files should be saved. After downloading these Debian files alien can convert them to a different format (e.g., rpm).

Installing using Flatpak

A flatpak bundle is available. Please download cvrdecode.flatpak. This is a Google Drive link. It may display a download button near the upper-left of the screen. When pressed, you may see a blank window for a few seconds. Then it will ask you what to do. You should save the file instead of running it (the link is not to an executable file). If you see a 'spinning' icon in the center of the screen, look for the download button and push it. You will have to use the 'back' button to return to this page.

For security, run a command that will generate a SHA-256 digest: 


  shasum -a 256 cvrdecode.flatpak
from the download directory. The digest printed by this command should be 

  f15f900eca5865b906dabadf93624185983011726d88775c68479f200d335d03
if the correct file was downloaded. 


  flatpack install cvrdecode.flatpak
for a system-wide installation, or 

  flatpack install --user cvrdecode.flatpak
to install for use by a single user (system privileges will not be needed in this case).

Installing using cvrdecode-install.jar

An installer is also available. Please download cvrdecode-install.jar. This is a Google Drive link. It may display a download button near the upper-left of the screen. When pressed, you may see a blank window for a few seconds. Then it will ask you what to do. You should save the file instead of running it (the link is not to an executable file). If you see a 'spinning' icon in the center of the screen, look for the download button and push it. You will have to use the 'back' button to return to this page.

For security, run a command that will generate a SHA-256 digest: 


  shasum -a 256 cvrdecode-install.jar
from the download directory. The digest printed by this command should be 

  91c7f82f62b94c1befdd98d77293ab8ee8a66807c829ef67b32c4262c0bc6f2b
if the correct file was downloaded.

Java 11 (or later) must be installed before the installer can be used. To start the installer, run the command 


  sudo java -jar cvrdecode-install.jar
and an installer start that will provide you with several installation options. These options let you pick the location where the software will be installed (generally used standard locations are shown). If sudo is not available, use whatever procedure grants you the privileges needed to install software. If the dry run option is checked, the installer will open a dialog box with a text field containing the corresponding command to enter in a terminal window given the options that were selected.

The command 


  java -jar cvrdecode-install.jar --help
will print all the command-line options. One in particular, --rootdir, can be used to install the software in a specified directory (e.g., for testing), in which case the use of sudo may not be necessary.

Some additional manual configuration is currently needed. The installation for Linux/Unix/MacOS should put a command named cvrdecode in a standard location and provide a man page, but will not install icons or integrate the program with one's desktop. Similar constraints apply to Windows systems and in particular, the installer does not do anything to the registry. If these additional steps are not performed, the program should still work normally, but may have to be started from a terminal.

These additional steps, while system dependent, should be trivial. For example, the additional steps that apt-get install cvrdecode will perform are to

  1. copy cvrdecode.desktop to an appropriate location.
  2. copy icons to the appropriate locations (the icons will be stored in a ZIP file named share.zip).
  3. run the command 
    
  update-mime-database
    in order to make the icons accessible without needing to reboot.
While easy, these steps still have to be tested, which requires access to systems the author does not have available.
