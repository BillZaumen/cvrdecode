CVRDecode Installation

There are three options for installing CVRDecode:

  • from Debian packages.
  • using flatpak.
  • using an installer program.

Installing from Debian packages

First add the Debian repository that contains the libbzdev-java and cvrdecode packages. Instructions are available at https://billzaumen.github.io/bzdev/ . The procedure involves downloading a GPG public key and copying it to the directory /etc/apt/trusted.gpg.d/. The next step is to add the line 

deb https://billzaumen.github.io/bzdev/archive/ hirsute contrib
to a new file placed in the directory /etc/apt/sources.list.d/. The final step is to run the command sudo apt update so that apt recognizes the newly added repository.

Next install the default Java for your system and run the following commands (as needed - some packages may already be installed):

  • sudo apt-get install libjcommander-java
  • sudo apt-get install libzxing-java
  • sudo apt-get install libzxing-core-java
  • sudo apt-get install libzxing-javase-java
  • sudo apt-get install libbzdev-java
  • sudo apt-get install cvrdecode

Installing using cvrdecode-install.jar

An installer is also available. Please download cvrdecode-install.jar. This is a Google Drive link. It may display a download button near the upper-left of the screen. When pressed, you may see a blank window for a few seconds. Then it will ask you what to do. You should save the file instead of running it (the link is not to an executable file). If you see a 'spinning' icon in the center of the screen, look for the download button and push it. You will have to use the 'back' button to return to this page.

For security, run a command that will generate a SHA-256 digest: 

shasum -a 256 cvrdecode-install.jar
from the download directory. The hash printed by this command should be 
b00a5c7cf639158c77841b849a7eb633e76b0be87aa929e59733caee1aad76d9
if the correct file was downloaded.

Java 11 must be installed before the installer, and the program, can be used. To start the installer, run the command 

sudo java -jar cvrdecode-install.jar
and a program will start that will provide you with several installation options. These options let you pick the location where the software will be installed (generally used standard locations are shown). If sudo is not available, use whatever procedure grants you the privileges needed to install software.

Some additional manual configuration is currently needed. The installation for Linux/Unix/MacOS should put a command named cvrdecode in a standard location and provide a man page, but will not install icons or integrate the program with one's desktop. Similar constraints apply to Windows systems and in particular, the installer does not do anything to the registry. If these additional steps are not performed, the program should still work normally, but may have to be started from a terminal.

These additional steps should be trivial. For example, the additional steps the Debian package will perform are to

  1. copy cvrdecode.desktop to an appropriate location.
  2. copy icons to the appropriate locations.
  3. run the command 
    update-mime-database
    in order to make the icons accessible without needing to reboot.
While easy, these steps still have to be tested, which requires access to systems the author does not have available.
